# 产测系统 MQTT 协议文档

## 1. 系统概述

本产测系统用于一款带云台的智能摄像头设备（om70X0X 系列）。设备端运行产测守护进程（client ID: `prodTest`），通过本地 MQTT broker 与上位机产测工具进行通信。上位机下发测试指令，设备端执行硬件测试并回报结果。

**MQTT 连接参数：**

| 参数 | 值 |
|------|-----|
| Broker 地址 | localhost |
| 端口 | 1883 |
| Client ID | prodTest |
| Keep Alive | 60 秒 |
| QoS | 2（所有消息） |
| 最大重连次数 | 10 |

---

## 2. 通用协议格式

### 2.1 Topic 命名规则

- 后缀 `R`（Request）：上位机 → 设备（下发指令）
- 后缀 `A`（Answer）：设备 → 上位机（回报结果）
- 数字编号 `10~19`：板级测试（PCBA 阶段）
- 数字编号 `20~29`：系统级测试（整机阶段）
- 数字编号 `30~39`：老化测试及辅助功能
- 数字编号 `50`：关机

### 2.2 通用请求 Payload（上位机 → 设备）

总长度：**38 字节**

| 偏移（字节） | 长度（字节） | 字段名 | 类型 | 说明 |
|:---:|:---:|--------|------|------|
| 0 | 16 | SN | ASCII 字符串 | 设备序列号 |
| 16 | 16 | bizid | ASCII 字符串 | 业务 ID |
| 32 | 8 | msgid | ASCII 字符串 | 消息 ID |

设备收到后会将 SN 写入 `/device_data/pcba.txt` 持久保存（若已存在相同 SN 则跳过写入）。

### 2.3 通用响应 Payload（设备 → 上位机）

总长度：**42 字节**

| 偏移（字节） | 长度（字节） | 字段名 | 类型 | 说明 |
|:---:|:---:|--------|------|------|
| 0 | 16 | SN | ASCII 字符串 | 原样回传 |
| 16 | 16 | bizid | ASCII 字符串 | 原样回传 |
| 32 | 8 | msgid | ASCII 字符串 | 原样回传 |
| 40 | 4 | error_code | uint32, 大端序（网络字节序） | 0 = 全部通过；非零 = 按位表示各错误项 |

**error_code 采用位掩码设计**，多个错误可以叠加。例如 `error_code = 0x00000003` 表示 bit0 和 bit1 对应的两项测试同时失败。

## 3. 板级测试（PCBA 阶段）

板级测试在 PCBA 裸板阶段执行，测试各核心硬件模块是否焊接正常。

### 3.1 RTC 时钟 + IC 检测（Topic: `10R` → `10A`）

**测试内容：** RTC 时钟芯片、充电 IC（I2C 地址 0x75）、电量计 IC（I2C 地址 0x38）

**error_code 位定义：**

| Bit | 值 | 含义 |
|:---:|------|------|
| 0 | 0x0001 | 打开 RTC 设备 `/dev/rtc` 失败 |
| 1 | 0x0002 | 读取 RTC 时间失败 |
| 2 | 0x0004 | RTC 年份早于 2026，时间无效 |
| 3 | 0x0008 | 读取系统时间失败 |
| 4 | 0x0010 | 转换本地时间失败 |
| 5 | 0x0020 | RTC 时间与系统时间不一致 |
| 6 | 0x0040 | 执行 I2C 查询命令失败 |
| 7 | 0x0080 | 充电 IC 检测失败（期望 0x01 或 0x00） |
| 8 | 0x0100 | 充电 IC 读取返回值失败 |
| 9 | 0x0200 | 电量计 IC 检测失败（期望 0xb1） |
| 10 | 0x0400 | 电量计 IC 读取返回值失败 |
| 11 | 0x0800 | 电量计设备 `/dev/om70X0X-bat` 不可读 |

---

### 3.2 麦克风测试（Topic: `11R` → `11A`）

**测试内容：** 设置 PDM1 增益 100%，蜂鸣器播放测试音，录制 5 秒 WAV 文件。

**测试流程：**
1. 删除旧录音文件 `/userdata/prod/test.wav`
2. 导出 GPIO 93 控制蜂鸣器
3. 设置麦克风增益（`amixer sset 'PDM1 Gain' 100%`）
4. 启动蜂鸣器线程播放测试音
5. 录音 5 秒（`arecord -D hw:0,0 -f S32_LE -r 48000 -c 2 -t wav -d 5`）
6. 回报结果

**error_code 位定义：**

| Bit | 值 | 含义 |
|:---:|------|------|
| 0 | 0x0001 | 设置麦克风增益失败 |
| 1 | 0x0002 | 录音命令执行失败 |

---

### 3.3 SoC 测试（Topic: `12R` → `12A`）

**测试内容：** 依次测试 GPU、DDR、CPU、NPU、eMMC 五个子系统。

**测试流程：**
1. GPU：设置 userspace governor，频率 900MHz，运行 `gpu_test`，检查负载是否达到 `100@`
2. DDR：设置 userspace governor，频率 2112MHz，运行 `stressapptest -s 5 -M 2048`，检查 `Status: PASS`
3. CPU：设置 performance governor，8 核压力测试 `stress --cpu 8 --timeout 5`，检查 `successful`
4. NPU：4 核矩阵乘法测试 `rknn_matmul_api_demo`，检查负载 `Core0: 99%, Core1: 99%`
5. eMMC：读写校验 `rwcheck`，检查 `rwcheck pass`

**error_code 位定义：**

| Bit | 值 | 含义 |
|:---:|------|------|
| 0 | 0x0001 | GPU 测试失败 |
| 1 | 0x0002 | DDR 测试失败 |
| 2 | 0x0004 | CPU 测试失败 |
| 3 | 0x0008 | NPU 测试失败 |
| 4 | 0x0010 | eMMC 测试失败 |

---

### 3.4 蓝牙测试（Topic: `13R`）

### 3.5 TF 卡测试（Topic: `14R` → `14A`）

**测试内容：** 请求 SD 卡性能数据，检查平均写入速度。

**测试流程：**
1. 设备向内部服务发送 `BFR`（Request SD Performance）请求，payload 为 1 字节（值 0x01）
2. 内部服务异步返回结果到 `BFA` topic
3. 设备解析平均写入速度（`avg_write_speed / 10` 即 MB/s）
4. 若速度 < 10 MB/s 或 err_code > 0，标记失败
5. 通过 `14A` 回报结果

**error_code 位定义：**

| Bit | 值 | 含义 |
|:---:|------|------|
| 0 | 0x0001 | TF 卡写入速度低于 10 MB/s |
| N | 1 << N | 内部服务返回的 err_code 对应位 |

---

### 3.8 按键测试（Topic: `17R` → `17A`）

**测试内容：** 监听 6 个按键输入，板级阶段只需任意 2 个按键按下即通过。超时 10 秒。

**被测按键：** KEY_POWER, KEY_F13, KEY_F14, KEY_F15, KEY_F16, KEY_F17

**error_code 位定义（初始值 0b111111 = 0x3F，按下一个键清除对应位）：**

| Bit | 值 | 含义 |
|:---:|------|------|
| 0 | 0x0001 | KEY_POWER 未检测到 |
| 1 | 0x0002 | KEY_F13 未检测到 |
| 2 | 0x0004 | KEY_F14 未检测到 |
| 3 | 0x0008 | KEY_F15 未检测到 |
| 4 | 0x0010 | KEY_F16 未检测到 |
| 5 | 0x0020 | KEY_F17 未检测到 |

**注意：** 板级阶段 `17A` 只在超时或 ≥2 个按键按下后发送一次。

### 3.9 2K 摄像头测试（Topic: `18R` → `18A`）

**测试内容：** 检查 gc4663 传感器 OTP 标定数据和二维码拍照功能。

**测试流程：**
1. 检查 OTP 文件 `/proc/otp_eeprom-4-50` 中 `RKAWBOTPParam` 和 `RKLSCOTPParam` 的 flag 是否为 1
2. 检查 gc4663 摄像头是否被 dmesg 检测到
3. 触发拍照（发送 `take_photo` topic），等待图片路径回传（`photo` topic）
4. 连续拍两张照片后回报结果

**error_code 位定义：**

| Bit | 值 | 含义 |
|:---:|------|------|
| 0 | 0x0001 | 发出拍照指令失败 |
| 1 | 0x0002 | 解析二维码失败 |
| 2 | 0x0004 | 摄像头未识别 |
| 3 | 0x0008 | OTP 标定校验失败 |

## 4. 系统级测试（整机阶段）

系统级测试在整机组装后执行，覆盖更多外设和交互功能。

### 4.1 按键测试（Topic: `20R` → `20A`）

与板级按键测试相同硬件，但**需全部 6 个按键按下才通过**。并且每按下一个键就实时回报一次当前 error_code，方便上位机实时显示按键状态。

error_code 位定义与 3.8 一致。

---

### 4.2 垂直霍尔传感器测试（Topic: `21R` → `21A`）

**测试内容：** 驱动俯仰电机往两个方向运动直到触发上下限位霍尔开关。

**测试流程：**
1. 向下运动直到触发下限位霍尔（超时 10 秒）
2. 向上运动直到触发上限位霍尔（超时 10 秒）

**error_code 位定义：**

| Bit | 值 | 含义 |
|:---:|------|------|
| 0 | 0x0001 | 上限位霍尔未检测到 |
| 1 | 0x0002 | 下限位霍尔未检测到 |

## 5. 老化测试

### 5.1 全量老化测试（Topic: `30R`）

**说明：** 这是最复杂的测试项，同时启动多个子系统进行长时间稳定性验证。

**测试流程：**
1. LED 切换为测试模式（白灯闪）
2. 写入 `2` 到 `/userdata/aging_completed.txt` 表示正在老化
3. 启动 track 跟踪模式
4. 启动录像
5. 启动老化主线程（周期性检测 WiFi 和摄像头状态）
6. 启动电池测试脚本（`battery-testaddbat.sh`）
7. 启动老化电机监控线程（监控时长和电量）
8. 启动水平电机循环测试线程（含霍尔校准）
9. 启动垂直电机循环测试线程

**老化时长：** 从 `/userdata/aging_time.conf` 读取（单位：小时），默认 4 小时。设为 `inf` 表示无限。

**周期性检测（每 10 秒一次）：**
- WiFi 扫描功能是否正常
- 摄像头 MIPI CSI-2 是否有错误日志
- 电池电量是否低于 5%

**老化完成标志（`/userdata/aging_completed.txt`）：**

| 值 | 含义 |
|-----|------|
| 1 | 老化正常完成 |
| 2 | 老化进行中 |
| 3 | 电机超时（预留，当前注释） |
| 30 | 上位机标记失败 |
| 31 | 电池电量过低 |
| 32 | WiFi 模块异常 |
| 33 | 霍尔电压校准失败 |
| 34 | 摄像头 MIPI 异常 |
| 35 | 霍尔管理器初始化失败 |

**老化结果 JSON 文件**（`/userdata/aging_result.json`）：

```json
{
  "sn": "XXXXXXXXXXXXXX",
  "test_item": "aging",
  "result": "pass",
  "fail_reason": "WIFI",
  "timestamp": 1700000000
}
```

`fail_reason` 可能值：`WIFI`、`CAM`、`HALL_VOLT`、`HALL_INIT`、`LOW_BAT`。

---

### 5.2 老化电机专项测试（Topic: `32R`）

仅启动电机老化测试（不含 WiFi、摄像头检测、录像等），启动水平/垂直电机循环测试。

### 5.3 老化停止（Topic: `34R`）

**处理逻辑：**
1. LED 切换回正常模式（红白交替）
2. 写入 `1` 到 `/userdata/aging_completed.txt`
3. 杀掉电池测试脚本
4. 置 `g_aging_motor_test_start = false` 和 `g_aging_test_start = false`

---

### 5.4 老化失败（Topic: `36R`）

**处理逻辑：**
1. LED 切换为测试失败模式（红灯闪）
2. 写入 `30` 到 `/userdata/aging_completed.txt`
3. 杀掉电池测试脚本
4. 停止所有老化线程

## 6. 辅助功能

### 6.1 心跳（Topic: `31R` → `31A`）

Payload 原样回传，无固定格式。用于上位机检测设备在线状态。

---

### 6.2 WiFi 连接（Topic: `37R` → `37A`）

**请求 Payload（非通用格式）：** ASCII 字符串

```
{SSID}--wifi--{PASSWORD}
```

**响应 Payload（`37A`）：** ASCII 字符串，为获取到的 IP 地址（如 `192.168.1.100`）。

**流程：** 调用 `wifi-connect.sh` 连接 WiFi，然后轮询 `wpa_cli status` 获取 IP，最多重试 20 次（每次间隔 3 秒）。

---

### 6.3 电池状态监听（Topic: `AJR`）

**Payload（非通用格式）：** 2 字节

| 偏移 | 长度 | 字段 | 说明 |
|:---:|:---:|------|------|
| 0 | 1 | capacity | 电池电量百分比（0~100） |
| 1 | 1 | charge_status | 充电状态 |

**设备端行为：**
- 监控电量，≤5% 标记低电量
- 断充续充管理：电量 ≥80% 时通过 I2C 停止充电（`i2cset -f -y 6 0x75 0x01 0x00`），≤70% 时恢复充电（`i2cset -f -y 6 0x75 0x01 0x01`）
- 无回报 topic

---

### 6.4 充电器状态（Topic: `CHANGER` + `38R` → `38A`）

**`CHANGER` topic：** 1 字节 payload，设备缓存充电器状态。

**`38R` topic：** 上位机请求上报充电器状态。

**`38A` 响应 Payload：** 43 字节

| 偏移 | 长度 | 字段 | 说明 |
|:---:|:---:|------|------|
| 0~37 | 38 | 通用头 | SN + bizid + msgid |
| 38 | 4 | error_code | 固定为 0 |
| 42 | 1 | changer_status | 充电器状态值 |

---

### 6.5 版本信息请求（Topic: `AZR`）

**Payload：** 1 字节，值为 `0x01`。

**说明：** 设备端主动发起，请求其他服务返回版本信息到 `AZA` topic。仅在 SN 已写入时触发。

---

### 6.6 关机（Topic: `50R`）

**处理逻辑：**
1. 停止所有老化测试标志
2. 取消老化电机测试线程
3. 执行关机脚本 `timeout 30s /bin/bash /oem/usr/scripts/power_off.sh`

---

### 6.7 录像状态通知（Topic: `AMR`）

**说明：** 设备订阅此 topic 监听录像状态变化，仅打印日志，不回报。

**状态码：** 0 = 录制中, 1 = 已暂停, 2 = 未录制

**错误码：**

| 值 | 含义 |
|------|------|
| 0 | 成功 |
| 1000 | SD 卡存储不足 |
| 1001 | SD 卡未插入 |
| 1002 | SD 卡未识别需格式化 |
| 1003 | SD 卡写入错误 |
| 1010 | 电量不足 |
| 1020 | 方向锁定中 |
| 1030 | 操作忙 |
| 1040 | 正在关机 |
| 1050 | 消息错误 |
| 1060 | AI 分析录像需标记站点 |
| 1070 | 电子围栏录像需标记 |

## 8. LED 状态指示

设备通过红灯（GPIO 144）和白灯（GPIO 89）指示当前状态：

| 模式 | 红灯 | 白灯 | 含义 |
|------|------|------|------|
| MODE_NORMAL | 交替闪 | 交替闪 | 待机，等待指令 |
| MODE_TEST | 灭 | 闪烁（1s 周期） | 测试/老化进行中 |
| MODE_TEST_DONE | 灭 | 常亮 | 老化测试通过 |
| MODE_TEST_FAIL | 闪烁（1s 周期） | 灭 | 老化测试失败 |

## 9. 典型交互时序

### 9.1 单项板级测试（以 SoC 测试为例）

```
上位机                                  设备
  │                                      │
  │─── [12R] SN+bizid+msgid ───────────>│
  │                                      │ 写入 SN

  │<── [12A] SN+bizid+msgid+error_code ─│
  │                                      │
```

### 9.2 老化测试完整流程

```
上位机                                  设备
  │                                      │
  │─── [30R] SN+bizid+msgid ───────────>│
  │                                      │ LED → 白灯闪
  │                                      │ 启动录像+电机+WiFi/CAM 检测
  │                                      │
  │                                      │ ... 持续数小时 ...
  │                                      │
  │    （正常完成）                       │ LED → 白灯常亮

  │─── [34R] 老化停止 ────────────────>│ LED → 红白交替
  │    或                                │
  │─── [36R] 老化失败 ────────────────>│ LED → 红灯闪
  │                                      │
```
